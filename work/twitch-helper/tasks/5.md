# Task 5: Twitch API Client + EventSub

## Description

REST клиент для Channel Points API (создание/список/fulfill/cancel наград) и EventSub WebSocket клиент (подписка на redemption-события, keepalive watchdog, реконнект). Это центральный компонент — от него зависят Queue, MaskAction и MediaAction.

## What to do

1. Создать `src/main/twitch/api.ts` — `TwitchApiClient`:
   - `createReward(title, cost, cooldownSeconds)` → POST `/helix/channel_points/custom_rewards`; 400 name conflict → throw с понятным сообщением
   - `listRewards()` → GET `/helix/channel_points/custom_rewards?only_manageable_rewards=true`
   - `deleteReward(rewardId)` → DELETE `/helix/channel_points/custom_rewards`
   - `fulfillRedemption(rewardId, redemptionId)` → PATCH status=FULFILLED
   - `cancelRedemption(rewardId, redemptionId)` → PATCH status=CANCELED
   - Все методы: при 401 → попытаться refresh токена; при неудаче → emit `auth:logout`
   - Атомарное создание награды: если config.write() упал — вызвать `deleteReward(newRewardId)` для rollback
2. Создать `src/main/twitch/eventsub.ts` — `EventSubClient`:
   - Подключиться к `wss://eventsub.wss.twitch.tv/ws`
   - При `session_welcome` → подписаться на `channel.channel_points_custom_reward_redemption.add` через REST
   - При `session_reconnect` → открыть новое соединение по reconnect_url, подписаться, закрыть старое
   - При `notification` → emit `redemption` событие с данными
   - Keepalive watchdog: если нет сообщений 60 секунд → закрыть соединение, переподключиться
3. Создать `src/main/twitch/client.ts` — `TwitchClient` (фасад):
   - Объединяет `TwitchApiClient` + `EventSubClient`
   - Emit `connected` / `disconnected` события
   - IPC push `twitch:status {connected: boolean}` при изменении состояния
   - `start()` / `stop()` методы для lifecycle

## Details

**Files:**
- `src/main/twitch/api.ts` — REST клиент для Channel Points API
- `src/main/twitch/eventsub.ts` — WebSocket EventSub клиент
- `src/main/twitch/client.ts` — фасад, объединяющий api + eventsub

**Dependencies:** ws (WebSocket клиент), AuthStore (Task 2)

**Edge cases:**
- Twitch EventSub keepalive_timeout_seconds приходит в session_welcome — использовать его для watchdog (обычно 10s, но может меняться)
- При reconnect: subscription POST должен использовать токен broadcaster_id из config
- Orphan reward: если `createReward` прошёл но последующий `config.write()` упал — удалить созданную награду

**Implementation hints:**
- EventSub subscription: POST `https://api.twitch.tv/helix/eventsub/subscriptions` с `transport.session_id`
- `broadcaster_id` = `config.broadcasterId` (из Task 4 OAuth flow)
- WS нужен как клиент для Twitch и как сервер для overlay — это разные экземпляры
