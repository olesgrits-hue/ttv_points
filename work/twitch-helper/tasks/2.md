# Task 2: Config & Auth Store

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Реализуем типизированное хранилище конфига (`config.json`) с атомарными записями и хранилище OAuth-токенов через Windows Credential Manager (keytar). `SlotService` инкапсулирует бизнес-логику управления слотами с ограничением в 5 штук.

Всё остальное приложение зависит от этого слоя данных.

## What to do

1. Создать `src/main/store/types.ts` — все TypeScript интерфейсы: `Config`, `Slot`, `BaseSlot`, `MaskSlot`, `MediaSlot`, `MemeSlot`, `LogEntry`, `MediaRegistryEntry`
2. Создать `src/main/store/config.ts` — `ConfigStore` класс:
   - Определять путь: `process.env.PORTABLE_EXECUTABLE_DIR` если `app.isPackaged`, иначе `process.cwd()`
   - `read()`: читать `config.json`, при отсутствии/поврежденном JSON возвращать defaults
   - `write(config)`: атомарная запись — сначала `.tmp`, потом rename
   - Defaults: `{ slots: [], removeMaskHotkey: '' }`
3. Создать `src/main/store/auth.ts` — `AuthStore` класс:
   - `saveTokens(access, refresh)`: keytar.setPassword для обоих
   - `getTokens()`: keytar.getPassword для обоих
   - `deleteTokens()`: keytar.deletePassword для обоих
   - Dev fallback: если keytar недоступен — читать `TWITCH_HELPER_ACCESS_TOKEN` из env
   - `isTokenExpired(config)`: проверять `config.tokenExpiresAt`
4. Создать `src/main/slots/service.ts` — `SlotService`:
   - `addSlot(slot)`: проверять `slots.length < 5`, добавлять, сохранять
   - `removeSlot(id)`: удалять по id, сохранять
   - `toggleSlot(id, enabled)`: обновлять enabled, сохранять
   - `getSlots()`: читать из config
5. Написать unit-тесты перед реализацией (TDD)

## Details

**Files:**
- `src/main/store/types.ts` — все интерфейсы данных
- `src/main/store/config.ts` — ConfigStore с атомарной записью
- `src/main/store/auth.ts` — AuthStore через keytar
- `src/main/slots/service.ts` — SlotService с max-5 enforcement

**Dependencies:** keytar v7.9.0 (Task 1 должна выполниться первой для rebuild)

**Edge cases:**
- `process.env.PORTABLE_EXECUTABLE_DIR` может быть undefined в dev → fallback на `process.cwd()`
- keytar может бросить ошибку если Windows Credential Manager недоступен → обработать gracefully
- `fs.rename` в Windows может падать cross-drive → использовать `fs.copyFile` + `fs.unlink` как fallback

**Implementation hints:**
- keytar service: `"twitch-helper"`, accounts: `"access_token"` и `"refresh_token"`
- Atomic write: `fs.writeFileSync(tmpPath, json)` → `fs.renameSync(tmpPath, finalPath)`
- Для тестов мокать keytar через `jest.mock('keytar')`
