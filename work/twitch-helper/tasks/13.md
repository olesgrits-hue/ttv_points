# Task 13: Security Audit

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:security-auditor` — [skills/security-auditor/SKILL.md](~/.claude/skills/security-auditor/SKILL.md)

## Description

Полный security-аудит всех компонентов приложения. Фокус на: хранение OAuth-токенов (keytar), валидация путей файлов (path traversal), nonce-аутентификация WebSocket, валидация IPC-входных данных, Electron security flags, привязка overlay-сервера к 127.0.0.1. Результат — аудит-отчёт с найденными проблемами по OWASP Top 10.

## What to do

1. Прочитать все файлы в `src/main/`, `src/renderer/`, `src/overlay/`
2. Проверить OAuth token storage:
   - Токены хранятся в keytar, не в plaintext файлах
   - `auth.json` не создаётся
   - config.json не содержит access_token / refresh_token
3. Проверить file path validation в `src/main/actions/media.ts`:
   - Блокируется ли path traversal (`../../../etc`)?
   - Валидируется ли путь относительно baseDir?
4. Проверить WebSocket nonce в `src/main/overlay/server.ts`:
   - Закрывает ли сервер соединения без аутентификации?
   - Nonce одноразовый (не переиспользуется)?
5. Проверить IPC validation в `src/main/ipc/`:
   - Все входные данные от renderer валидируются в main process?
   - Нет опасных операций без проверки типов и диапазонов?
6. Проверить Electron BrowserWindow в `src/main/window.ts`:
   - `contextIsolation: true`
   - `sandbox: true`
   - `nodeIntegration: false`
7. Проверить overlay server binding:
   - Сервер слушает на `127.0.0.1`, не `0.0.0.0`
8. Проверить PKCE flow в `src/main/twitch/auth.ts`:
   - code_verifier не логируется и не передаётся в renderer
   - state parameter проверяется при callback
9. Написать аудит-отчёт как финальный ответ
