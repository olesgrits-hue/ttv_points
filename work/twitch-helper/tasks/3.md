# Task 3: Overlay Server

## Description

Express HTTP-сервер на `127.0.0.1:7891` отдаёт прозрачную overlay-страницу для OBS Browser Source и медиафайлы через HTTP (OBS Chromium блокирует `file://`). WebSocket-сервер (ws) с nonce-аутентификацией принимает команды воспроизведения и получает сигналы окончания.

## What to do

1. Создать `src/main/overlay/registry.ts` — `MediaRegistry`:
   - `register(filePath): string` — генерирует uuid, сохраняет в Map, возвращает id
   - `resolve(id): string | undefined` — возвращает filePath по id
   - `deregister(id)` — удаляет запись
2. Создать `src/main/overlay/server.ts` — `OverlayServer`:
   - Express на `127.0.0.1:7891` (не 0.0.0.0)
   - `Cache-Control: no-store` на всех ответах
   - `GET /overlay` → отдаёт `src/overlay/index.html`
   - `GET /media/:id` → resolve из registry → отдаёт файл с правильным Content-Type; 404 если не найден
   - ws сервер на том же HTTP сервере (через `server.on('upgrade')`)
   - WS nonce: при подключении клиент должен отправить `{type:'auth', nonce}` первым; иначе закрыть соединение
   - `play(id, filePath): Promise<void>` — push `{type:'play', url:'/media/:id'}` в overlay, ждёт `playback_ended` или 120s таймаут
   - `generateNonce(): string` — генерирует одноразовый nonce для текущей сессии
3. Создать `src/overlay/index.html` — прозрачная HTML-страница:
   - `background: transparent` / `background-color: transparent`
   - JavaScript: подключение к WS, аутентификация nonce, обработка `play` команды
4. Создать `src/overlay/player.ts` — логика плеера:
   - При получении `{type:'play', url}` — создать `<video>` элемент
   - Случайная позиция (x, y) в пределах window bounds
   - Случайный угол поворота (-15° до +15°)
   - Фиксированный размер 400×300px
   - По завершению видео — отправить `{type:'playback_ended'}` через WS
5. Написать integration-тесты (supertest + ws клиент)

## Details

**Files:**
- `src/main/overlay/registry.ts` — in-memory Map для медиафайлов
- `src/main/overlay/server.ts` — Express + ws сервер
- `src/overlay/index.html` — прозрачная overlay-страница
- `src/overlay/player.ts` — видеоплеер с рандомной позицией/углом

**Dependencies:** express, ws, uuid (Task 1)

**Edge cases:**
- Порт 7891 может быть занят → логировать ошибку, не крашить всё приложение
- WS клиент может отключиться до `playback_ended` → resolve Promise по 120s таймауту
- Большие видеофайлы → использовать stream (res.sendFile или pipe) вместо readFileSync

**Implementation hints:**
- `server.listen(7891, '127.0.0.1', callback)`
- WS: `new WebSocket.Server({ server })` — shared HTTP server
- Для теста overlay-страницы: проверять строку `background: transparent` или `background-color: transparent` в HTML
- `res.sendFile(filePath)` автоматически определяет Content-Type по расширению
