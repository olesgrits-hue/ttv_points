# Task 15: Pre-deploy QA

## Required Skills

Before starting, load: `/skill:pre-deploy-qa` — [SKILL.md](../../../.claude/skills/pre-deploy-qa/SKILL.md) (if available)

## Description

Final acceptance gate before the twitch-helper feature is considered production-ready. This task runs the full test suite, verifies every acceptance criterion from both tech-spec and user-spec (agent-verifiable parts), and confirms that the built Electron app satisfies all security and correctness requirements.

The task depends on all three audit tasks (code, security, test) completing first so that any blocking findings are already resolved. No code is written here — the agent only executes, observes, and reports.

## What to do

1. Run the full test suite with `npm test`. Every unit and integration test must pass with zero failures.
2. Verify `config.json` is written to `PORTABLE_EXECUTABLE_DIR` — inspect the ConfigStore implementation and confirm the path resolution logic uses `process.env.PORTABLE_EXECUTABLE_DIR` when `app.isPackaged` is true, and falls back to `process.cwd()` in dev.
3. Verify OAuth tokens are stored in Windows Credential Manager via keytar — confirm no plaintext `auth.json` or `access_token` field exists in `config.json` after auth flow; confirm `keytar.setPassword` is used for `access_token` and `refresh_token`.
4. Start the overlay server and verify `GET http://127.0.0.1:7891/overlay` returns 200 with `background: transparent` in the body.
5. Register a test media file, verify `GET http://127.0.0.1:7891/media/:id` returns 200 with correct file bytes; then deregister and confirm 404.
6. Run the Twitch CLI EventSub integration test: trigger `channel-points-custom-reward-redemption-add` and confirm the subscription POST fires within 10 seconds of WebSocket connect.
7. Confirm FIFO queue behaviour from unit tests: items process in order; a 6th enqueue attempt returns an error without processing.
8. Confirm from unit tests: MaskAction fires remove-mask hotkey after exactly 30 seconds (fake timers); MediaAction cancels on empty folder; 120s timeout triggers fulfill when overlay WS is disconnected.
9. Confirm path traversal is blocked: a `filePath` containing `../../../etc` is rejected before the file is served.
10. Confirm overlay WS rejects unauthenticated connections (integration test passes).
11. Inspect `src/main/window.ts` (or equivalent BrowserWindow config) and confirm `contextIsolation: true`, `sandbox: true`, `nodeIntegration: false` are all set.
12. Write QA report to `c:/Cloude/SNAP CAM/work/twitch-helper/logs/working/task-15/qa-report.json` with pass/fail status for each criterion.

### Smoke

Start the overlay server (or run it as part of integration test harness) and probe the HTTP endpoints:

```bash
# Overlay page
curl -i http://127.0.0.1:7891/overlay
# Expected: HTTP/1.1 200 OK, body contains "background: transparent"

# Media endpoint — register a file first via the registry API or integration test helper, then:
curl -i http://127.0.0.1:7891/media/<registered-id>
# Expected: HTTP/1.1 200 OK, correct Content-Type, file bytes in body

# After deregister:
curl -i http://127.0.0.1:7891/media/<same-id>
# Expected: HTTP/1.1 404 Not Found

# Twitch CLI EventSub trigger (requires Twitch CLI installed and app running):
twitch event trigger channel-points-custom-reward-redemption-add
# Expected: subscription POST observed within 10s of WS connect (logged or asserted in integration test)
```

### Key implementation details to check

**Config path (Decision 5):**
`src/main/store/config.ts` must use:
```typescript
const base = app.isPackaged
  ? process.env.PORTABLE_EXECUTABLE_DIR!
  : process.cwd();
const configPath = path.join(base, 'config.json');
```
Atomic writes: write to `config.json.tmp`, then `fs.renameSync`.

**Auth token storage (Decision 4):**
`src/main/store/auth.ts` must call:
```typescript
await keytar.setPassword('twitch-helper', 'access_token', accessToken);
await keytar.setPassword('twitch-helper', 'refresh_token', refreshToken);
```
Non-sensitive metadata (`userId`, `broadcasterId`, `tokenExpiresAt`) may live in `config.json`.

**Electron security flags (Decision 9):**
`src/main/window.ts` BrowserWindow constructor must include:
```typescript
webPreferences: {
  contextIsolation: true,
  sandbox: true,
  nodeIntegration: false,
}
```

**Path traversal guard (MediaAction):**
`src/main/actions/media.ts` must validate that the resolved absolute path starts with the configured `filePath` or `folderPath` base — rejecting any path that escapes via `../` sequences.

**Overlay WS nonce auth:**
`src/main/overlay/server.ts` WebSocket `connection` handler must close the socket if the first message does not carry the correct nonce.

### Edge cases to confirm are tested

- Queue slot 6 (max is 5): `queue.enqueue()` on a full queue must return/throw an error — verify the unit test asserts this explicitly.
- 120s timeout path: the unit test must use fake timers to advance 120 000ms and assert `fulfillRedemption` was called without a `playback_ended` message arriving.
- Empty folder for MemeSlot: `MediaAction` must call `cancelRedemption` and emit a red log entry — confirm the unit test checks both.
