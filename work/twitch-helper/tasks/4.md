## Description

Реализуем OAuth 2.0 Authorization Code + PKCE поток для Twitch. На логине открываем браузер с callback-сервером на OS-назначенном порту. После авторизации сохраняем токены в keytar, метаданные в config.json. На старте проверяем истечение токена и пытаемся обновить; если не получается — показываем экран логина.

## What to do

1. Создать `src/main/twitch/auth.ts` — `TwitchAuth`:
   - `startLogin()`: генерировать PKCE verifier/challenge (pkce-challenge), запустить callback HTTP-сервер на `listen(0)` (OS-порт), открыть системный браузер на `https://id.twitch.tv/oauth2/authorize?...` со scopes `channel:read:redemptions channel:manage:redemptions`
   - Принять callback с `?code=...`, закрыть callback-сервер
   - POST `https://id.twitch.tv/oauth2/token` для обмена кода на токены
   - Сохранить access_token + refresh_token через `AuthStore`
   - Сохранить userId, broadcasterId, tokenExpiresAt в `ConfigStore`
2. Создать `src/main/twitch/auth.ts` — `refreshToken(refreshToken): Promise<Tokens>`:
   - POST `https://id.twitch.tv/oauth2/token` с `grant_type=refresh_token`
   - Обновить токены в AuthStore и config.json
   - При 400/401 — удалить токены, вернуть ошибку
3. Создать `src/main/ipc/auth.ts` — IPC handlers:
   - `ipcMain.handle('auth:login')` → вызвать `TwitchAuth.startLogin()`
   - `ipcMain.handle('auth:logout')` → `AuthStore.deleteTokens()`, очистить metadata в config
   - На старте приложения: если `config.tokenExpiresAt` истёк — попытаться refresh; если failed → emit `auth:logout` → renderer показывает auth screen
4. Dev fallback: если `process.env.TWITCH_HELPER_ACCESS_TOKEN` задан — пропустить PKCE flow, использовать напрямую

## Details

**Files:**
- `src/main/twitch/auth.ts` — PKCE flow, callback server, token exchange, refresh
- `src/main/ipc/auth.ts` — IPC handlers для renderer

**Dependencies:** pkce-challenge, AuthStore (Task 2), ConfigStore (Task 2)

**Edge cases:**
- Callback server: таймаут 5 минут если пользователь не авторизовался — закрыть сервер
- state parameter в OAuth URL — проверять при callback чтобы предотвратить CSRF
- broadcasterId может отличаться от userId для Affiliate аккаунтов — получать через `/helix/users`

**Implementation hints:**
- `pkce-challenge`: `const { code_verifier, code_challenge } = await pkceChallenge()`
- Callback URL: `http://localhost:${port}/callback` — включить в `redirect_uri`
- После получения кода — немедленно закрыть callback HTTP сервер
