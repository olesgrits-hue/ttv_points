# Task 9: Media Action Handler

## Required Skills

Before starting, load: /skill:code-writing — [SKILL.md](../../../.claude/skills/code-writing/SKILL.md)

## Description

Implements the `MediaAction` handler that executes media/meme slot redemptions. When a redemption is dispatched from the queue, this handler:

1. Resolves the target file — direct path for `media` slots, random pick from folder for `meme` slots
2. Validates the resolved path against the configured base path to block directory traversal attacks
3. Registers the file in `OverlayServer`'s in-memory media registry (gets a UUID-keyed `/media/:id` URL)
4. Pushes a `{type: 'play', url: '/media/:id', nonce}` command to the overlay via WebSocket
5. Awaits a `playback_ended` event from the overlay, with a 120-second timeout fallback when the overlay WebSocket is disconnected
6. Deregisters the file from the registry after playback completes (or times out)
7. Calls `fulfillRedemption` on success or `cancelRedemption` on error, then emits an IPC log entry

This is a Wave 3 task. It depends on Task 4 (OAuth/Twitch API client with fulfill/cancel), Task 5 (Twitch API client with fulfill/cancel REST calls), and Task 6 (no direct dependency, but the registry/server it uses is built in Task 3 of Wave 1). The handler sits at the intersection of the overlay server (registry + WebSocket push) and Twitch API (fulfill/cancel).

## What to do

1. Create `src/main/actions/media.ts` and export a `MediaAction` class (or function) that accepts injected dependencies: `OverlayServer` instance, `TwitchApiClient` instance, and `ConfigStore` instance.

2. Implement file resolution:
   - For `MediaSlot` (`type === 'media'`): use `slot.filePath` directly
   - For `MemeSlot` (`type === 'meme'`): read `slot.folderPath` with `fs.readdirSync` (or async equivalent), filter to files only, pick a random entry. If the folder is missing or empty — throw an error immediately (do not proceed to registry)

3. Implement path validation:
   - Resolve the candidate path to an absolute path via `path.resolve()`
   - Validate that it does not escape the configured base path (use `path.relative()` and check it does not start with `..`). The base path for media validation is the directory of `slot.filePath` / `slot.folderPath` as configured — or alternatively validate that the resolved path starts with the declared folder path for meme slots. For `media` slots, validate the path is exactly the configured absolute path (no traversal possible via symlink-like injection). The critical rule: if the resolved path does not equal `path.normalize(slot.filePath)` (for media) or does not start with `path.normalize(slot.folderPath) + path.sep` (for meme), reject it.
   - On validation failure: throw a traversal error (do not register in registry)

4. Implement registry + WebSocket push:
   - Call `overlayServer.registerMedia(filePath)` → returns `{ id, url }` (where `url = '/media/:id'`)
   - Push `{ type: 'play', url, nonce: overlayServer.nonce }` via `overlayServer.sendToOverlay(message)`
   - Wrap in try/finally to guarantee deregistration even on unexpected throws

5. Implement playback await with timeout:
   - Subscribe to the overlay server's `playback_ended` event (EventEmitter or Promise-based)
   - If the overlay WebSocket is not connected, start the 120-second timeout immediately
   - Use `Promise.race([playbackEndedPromise, timeoutPromise])` — timeout resolves (does not reject) so the action still fulfills after 120s

6. Implement cleanup and outcome reporting:
   - In the `finally` block: `overlayServer.deregisterMedia(id)`
   - On success (playback ended or timeout): call `twitchApi.fulfillRedemption(redemption.rewardId, redemption.id)`, emit IPC log entry with `status: 'success'`
   - On error (file not found, folder empty, validation failure): call `twitchApi.cancelRedemption(redemption.rewardId, redemption.id)`, emit IPC log entry with `status: 'error', errorMessage`

7. Write unit tests in `tests/actions/media.test.ts` (see TDD Anchor below) before or alongside implementation.

### File resolution

**MediaSlot (`type === 'media'`):**

const resolvedPath = path.normalize(slot.filePath);
// validate: resolvedPath must equal path.normalize(slot.filePath)
// (trivially true — but guards against future injection via slot mutation)

**MemeSlot (`type === 'meme'`):**
```typescript
const entries = await fs.promises.readdir(slot.folderPath, { withFileTypes: true });
const files = entries.filter(e => e.isFile()).map(e => path.join(slot.folderPath, e.name));
if (files.length === 0) throw new Error(`Meme folder is empty: ${slot.folderPath}`);
const resolvedPath = files[Math.floor(Math.random() * files.length)];
```

Handle `ENOENT` from `readdir` as "folder missing" error — same cancel+error path as empty folder.

### Path traversal validation

```typescript
function validatePath(resolvedPath: string, baseDir: string): void {
  const rel = path.relative(baseDir, resolvedPath);
  if (rel.startsWith('..') || path.isAbsolute(rel)) {
    throw new Error(`Path traversal blocked: ${resolvedPath}`);
  }
}
```

For `MediaSlot`, `baseDir` is `path.dirname(slot.filePath)`.
For `MemeSlot`, `baseDir` is `slot.folderPath`.

### 120-second timeout behavior

The timeout must **fulfill** (not cancel) the redemption — this is by design. When OBS/overlay is disconnected, the media was likely served and played; punishing the viewer with a cancel would be incorrect. The 120s is a safety net.

```typescript
const PLAYBACK_TIMEOUT_MS = 120_000;

const timeoutPromise = new Promise<'timeout'>(resolve =>
  setTimeout(() => resolve('timeout'), PLAYBACK_TIMEOUT_MS)
);
const result = await Promise.race([playbackDone.then(() => 'done' as const), timeoutPromise]);
// both 'done' and 'timeout' → fulfillRedemption
```

Use `jest.useFakeTimers()` in the timeout unit test. Ensure `jest.runAllTimers()` or `jest.advanceTimersByTime(120_000)` is called after setup.

### Error handling matrix

| Scenario | Registry | WS push | Outcome |
|----------|----------|---------|---------|
| Folder missing/empty | Never registered | Not sent | `cancelRedemption` + error log |
| Path traversal blocked | Never registered | Not sent | `cancelRedemption` + error log |
| File found, overlay connected | Registered → deregistered | Sent | `fulfillRedemption` + success log |
| File found, overlay disconnected | Registered → deregistered | Skipped or queued | `fulfillRedemption` after 120s + success log |
| Registry throws unexpectedly | Never registered | Not sent | `cancelRedemption` + error log |

### Edge cases

- **File deleted between config save and redemption:** `fs.promises.readdir` or `path.resolve` will succeed, but the HTTP server will get a read error when OBS fetches `/media/:id`. Handle at the registry-serve level (404) and consider emitting a deferred error log if possible. At minimum, `MediaAction` validates the file exists via `fs.promises.access(resolvedPath)` before registering.
- **Concurrent redemptions:** The queue processes one at a time (FIFO from Task 7), so `MediaAction` is never called concurrently. No locking needed.
- **WS send fails (overlay not connected):** `sendToOverlay` should not throw — it should silently skip if no client is connected. The 120s timeout handles this case.
