## Description

## Details

**Five high-priority areas to verify first (from tech-spec description):**

1. **MaskAction fake-timer sequence** ÔÇö The test must use Jest fake timers (`jest.useFakeTimers()`). It must call `MaskAction.execute(...)`, then call `jest.advanceTimersByTime(30000)`, then assert: (a) apply-mask hotkey was called first, (b) remove-mask hotkey was called after the advance, (c) `fulfillRedemption` was called exactly once. A test that only checks the hotkey calls but not `fulfillRedemption` ordering is `major`.

2. **TwitchClient reconnect re-subscribe** ÔÇö The mocked WS must simulate a close event, a re-connect, and a second `session_welcome` with a *different* `session_id`. The test must assert that the subscription POST is made with the new `session_id`, not the stale one. A test that only asserts "subscribe was called twice" without checking the `session_id` value is `major`.

3. **Disabled-slot cancel flow** ÔÇö The test must enqueue a redemption whose `rewardId` maps to a slot with `enabled: false`. It must assert that `cancelRedemption` was called and that neither `MaskAction.execute` nor `MediaAction.execute` was called. Missing entirely is `critical`.

4. **Token expiry/refresh unit tests** ÔÇö `AuthStore` startup must detect `tokenExpiresAt` in the past, call `refreshToken`, and on refresh failure call `keytar.deletePassword` and emit `auth:logout` IPC. The test must assert that no Twitch API call occurs before the expiry check. Missing is `critical`.

5. **Keepalive watchdog** ÔÇö The test must use fake timers, advance by 60s without any message on the mocked WS, and assert: (a) `ws.close()` was called, (b) a new WS connection was opened (reconnect). Missing is `critical`.

**Edge cases to look for:**
- Tests that mock `keytar` globally but don't restore mocks between tests ÔÇö can cause cross-test contamination
- MaskAction tests that use real `setTimeout` instead of fake timers ÔÇö will cause test suite to hang or time out
- Integration tests that bind to port 7891 without releasing it ÔÇö can cause port-conflict failures on re-run
- Tests that import robotjs directly rather than mocking it ÔÇö will fail in CI without the native binary
