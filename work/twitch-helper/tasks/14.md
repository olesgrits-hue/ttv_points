## Description

Full-feature test quality audit for the twitch-helper feature. By wave 6, all code tasks (1вЂ“11) are complete and the test suite exists. This task audits the completeness and correctness of that suite against the testing strategy defined in the tech-spec.

The audit focuses on five specific risk areas identified in the tech-spec: MaskAction fake-timer sequence, TwitchClient reconnect re-subscribe, disabled-slot cancel flow, token expiry/refresh unit tests, and keepalive watchdog. Beyond these, the audit covers the full unit and integration test inventory to identify any missing, incorrect, or misleading tests.

The output is a structured audit report saved to `work/twitch-helper/logs/working/task-14/test-audit.json`.

## What to do

1. Read all existing test files under `tests/` (unit and integration).
2. For each unit test area from the tech-spec strategy, verify coverage is present and adequate:
   - **Queue:** FIFO order, max-5-slots (6th enqueue returns error), disabled-slot cancel (no action fires), error propagation
   - **MaskAction:** full fake-timer sequence вЂ” mask hotkey fires в†’ `advanceTimersByTime(30000)` в†’ remove-mask hotkey fires в†’ `fulfillRedemption` called in correct order; cancel on robotjs error
   - **MediaAction:** specific file resolved; random file from non-empty folder; empty folder triggers cancel+refund; 120s timeout triggers fulfill when overlay WS disconnected; path traversal attempt (`../../../etc`) blocked
   - **ConfigStore:** read/write round-trip; missing file returns defaults; corrupt JSON returns defaults; slot count в‰Ґ6 rejected
   - **TwitchClient (mocked WS):** EventSub message parsing; reconnect fires re-subscribe with new `session_id`; 60s keepalive watchdog closes and reopens connection; 401 triggers refresh attempt; failed refresh emits `auth:logout`
   - **AuthStore:** token save/read via keytar mock; expired token detected on startup в†’ `auth:logout` IPC before any API call
   - **HotkeyParser:** valid combos (e.g. `"ctrl+shift+1"`); empty string в†’ error; unknown key name в†’ error
3. For each integration test area, verify coverage:
   - Overlay HTTP: `GET /overlay` в†’ 200, `Content-Type: text/html`, `background: transparent`
   - Media serving: register file в†’ `GET /media/:id` в†’ 200, correct Content-Type, file bytes
   - Overlay WebSocket: connect with valid nonce в†’ send `play` в†’ client receives; send `playback_ended` в†’ server resolves promise
   - WS unauthenticated: connect without nonce в†’ first non-auth message в†’ server closes connection
   - Twitch EventSub flow: connect в†’ receive `session_welcome` в†’ subscription POST fires within 10s
   - Config round-trip: save 5 slots в†’ restart ConfigStore instance в†’ all slots restored
4. For each gap or defect found, categorize it by severity: `critical` (test missing for a core acceptance criterion), `major` (test exists but does not assert the right thing), `minor` (test name misleading or assertion incomplete).
5. Write the audit report to `work/twitch-helper/logs/working/task-14/test-audit.json`.
6. If critical or major gaps are found, write the missing or corrected tests directly in the appropriate test files to close those gaps.

## Details

**Five high-priority areas to verify first (from tech-spec description):**

1. **MaskAction fake-timer sequence** вЂ” The test must use Jest fake timers (`jest.useFakeTimers()`). It must call `MaskAction.execute(...)`, then call `jest.advanceTimersByTime(30000)`, then assert: (a) apply-mask hotkey was called first, (b) remove-mask hotkey was called after the advance, (c) `fulfillRedemption` was called exactly once. A test that only checks the hotkey calls but not `fulfillRedemption` ordering is `major`.

2. **TwitchClient reconnect re-subscribe** вЂ” The mocked WS must simulate a close event, a re-connect, and a second `session_welcome` with a *different* `session_id`. The test must assert that the subscription POST is made with the new `session_id`, not the stale one. A test that only asserts "subscribe was called twice" without checking the `session_id` value is `major`.

3. **Disabled-slot cancel flow** вЂ” The test must enqueue a redemption whose `rewardId` maps to a slot with `enabled: false`. It must assert that `cancelRedemption` was called and that neither `MaskAction.execute` nor `MediaAction.execute` was called. Missing entirely is `critical`.

4. **Token expiry/refresh unit tests** вЂ” `AuthStore` startup must detect `tokenExpiresAt` in the past, call `refreshToken`, and on refresh failure call `keytar.deletePassword` and emit `auth:logout` IPC. The test must assert that no Twitch API call occurs before the expiry check. Missing is `critical`.

5. **Keepalive watchdog** вЂ” The test must use fake timers, advance by 60s without any message on the mocked WS, and assert: (a) `ws.close()` was called, (b) a new WS connection was opened (reconnect). Missing is `critical`.

**Edge cases to look for:**
- Tests that mock `keytar` globally but don't restore mocks between tests вЂ” can cause cross-test contamination
- MaskAction tests that use real `setTimeout` instead of fake timers вЂ” will cause test suite to hang or time out
- Integration tests that bind to port 7891 without releasing it вЂ” can cause port-conflict failures on re-run
- Tests that import robotjs directly rather than mocking it вЂ” will fail in CI without the native binary
