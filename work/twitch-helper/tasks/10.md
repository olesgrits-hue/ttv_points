# Task 10: Main UI — Auth Screen, Layout, Event Log

## Required Skills

Before starting, load: /skill:code-writing — [SKILL.md](../../../.claude/skills/code-writing/SKILL.md)

## Description

Implements the React renderer layer of the Electron app. Configures `BrowserWindow` in `src/main/window.ts` with the required security flags (`contextIsolation: true`, `sandbox: true`, `nodeIntegration: false`). Builds two screens — `AuthScreen` (single login button that triggers the OAuth IPC call) and `MainScreen` (two-section layout: Snap Camera + Media, OBS URL display, connection status banner, event log). Builds the `EventLog` component that renders in-memory log entries in `[time] [viewer] [reward] 🟢/🔴` format with an error tooltip on hover.

This task runs in Wave 4, parallel to Wave 3 action handlers. It depends on Task 4 (OAuth IPC), Task 5 (Twitch status IPC), and Task 6 (snap-camera-server, for context on how IPC channels are named). The slot management UI (Task 11) will build on top of `MainScreen` created here.

## What to do

1. **Configure BrowserWindow** in `src/main/window.ts`: create and export a `createWindow()` function that instantiates `BrowserWindow` with `webPreferences: { contextIsolation: true, sandbox: true, nodeIntegration: false }`. Load the Vite dev server URL in dev mode or the built `index.html` in production. Set up IPC handlers to navigate between auth and main screens (listen for `auth:navigate` to switch screen state in renderer, or emit `auth:logout` to force return to auth screen).

2. **Build `AuthScreen`** in `src/renderer/screens/AuthScreen.tsx`: single screen with a "Войти через Twitch" button. On click, calls `window.electronAPI.login()` (via contextBridge preload). No other elements needed — keep it minimal.

3. **Build `MainScreen`** in `src/renderer/screens/MainScreen.tsx`: top-level layout with:
   - OBS URL text display: static string `http://127.0.0.1:7891/overlay` with a copy-to-clipboard button
   - Connection status banner: subscribes to `twitch:status` IPC events; shows "Соединение потеряно, переподключаюсь..." when `connected: false`; hides when `connected: true`
   - Two named sections: "Snap Camera" and "Media" — empty placeholders for now (slot cards arrive in Task 11)
   - `EventLog` component at the bottom

4. **Build `EventLog`** in `src/renderer/components/EventLog.tsx`: accepts `entries: LogEntry[]` prop. Renders each entry as `[HH:MM:SS] [viewerName] [rewardTitle] 🟢` or `[HH:MM:SS] [viewerName] [rewardTitle] 🔴`. On hover of 🔴 shows a tooltip with `errorMessage`. Entries are ordered newest-first. The component receives entries from `MainScreen` which maintains the list in React state, updated via `twitch:log` IPC event.

5. **Wire `App.tsx`**: root component that holds `screen: 'auth' | 'main'` state. Renders `<AuthScreen>` or `<MainScreen>` based on state. Subscribes to `auth:logout` IPC event to switch back to auth screen. On mount, checks if already authenticated (call IPC `auth:check`) and navigates to main screen if so.

6. **Expose IPC via preload script** (create or update `src/main/preload.ts`): use `contextBridge.exposeInMainWorld('electronAPI', {...})` to expose:
   - `login()` — invokes `auth:login` IPC
   - `onAuthLogout(cb)` — subscribes to `auth:logout` IPC
   - `onTwitchStatus(cb)` — subscribes to `twitch:status` IPC
   - `onLogEntry(cb)` — subscribes to `twitch:log` IPC
   - `checkAuth()` — invokes `auth:check` IPC, returns `boolean`
   Point `BrowserWindow` preload option to this file.

- All listeners must return a cleanup function (call `ipcRenderer.removeListener`) so React effects can unsubscribe on unmount

**`src/renderer/App.tsx`**
- `useState<'auth' | 'main'>('auth')`

**`src/renderer/screens/AuthScreen.tsx`**

- `useEffect`: subscribe to `twitch:status` → update connected state; subscribe to `twitch:log` → prepend entry to list (newest first); return cleanup for both
- Render: OBS URL text + copy button; connection banner (conditional); two sections ("Snap Camera", "Media") as labeled containers; `<EventLog entries={entries} />`

**`src/renderer/components/EventLog.tsx`**

- Format timestamp: `new Date(entry.timestamp).toLocaleTimeString('ru-RU', { hour12: false })`
- Status icon: `entry.status === 'success' ? '🟢' : '🔴'`
- Error tooltip: wrap 🔴 in a `<span title={entry.errorMessage}>` or a custom CSS tooltip (`position: relative; :hover::after { content: attr(data-tooltip) }`)

### IPC channel names (from tech-spec and Tasks 4/5)

| Channel | Direction | Payload |
|---|---|---|
| `auth:login` | renderer → main | (none) |
| `auth:check` | renderer → main (invoke) | returns `boolean` |
| `auth:logout` | main → renderer | (none) |
| `twitch:status` | main → renderer | `{ connected: boolean }` |
| `twitch:log` | main → renderer | `LogEntry` |

These channels must be consistent with the IPC handlers registered in Tasks 4 and 5. If those tasks use different names, align with them — do not introduce new channel names unilaterally.

### Electron security constraints

Because `sandbox: true` and `nodeIntegration: false` are set, the renderer has no access to Node.js APIs. All communication goes through the preload's `contextBridge`. Do not attempt to `require()` anything in renderer code. Import only from `src/renderer/` and shared type files that are bundled by Vite.

The preload script runs in a special context — it has access to `ipcRenderer` but must only expose a minimal, typed surface via `contextBridge`. Never expose `ipcRenderer` itself.

### Testing approach for renderer

Use `@testing-library/react` + `jsdom`. Mock `window.electronAPI` in test setup:

```ts
// tests/setup.ts or inline in each test
global.window.electronAPI = {
  login: jest.fn(),
  onAuthLogout: jest.fn(() => jest.fn()), // returns cleanup fn
  onTwitchStatus: jest.fn(() => jest.fn()),
  onLogEntry: jest.fn(() => jest.fn()),
  checkAuth: jest.fn().mockResolvedValue(false),
};
```

For testing IPC event callbacks: capture the callback passed to `onTwitchStatus` (via `jest.fn().mockImplementation(cb => { capturedCb = cb; return jest.fn(); })`) then call `capturedCb({ connected: false })` to simulate an event.

### Dependencies

- `react`, `react-dom` — from Task 1 scaffold
- `@testing-library/react`, `@testing-library/user-event` — for renderer tests (add to devDependencies if not already present from Task 1)
- No new runtime dependencies needed

### Edge cases

- Auth check on mount is async: show a brief loading state or simply render auth screen until the check resolves (to avoid flicker of wrong screen)
- Log entries list can grow unbounded in memory: cap at 200 entries (drop oldest) to avoid memory pressure during long streams
- `errorMessage` may be undefined for success entries: ensure tooltip renders only when `errorMessage` is truthy
- Cleanup IPC listeners on component unmount to prevent memory leaks and stale-closure callbacks after navigation
