# Task 6: snap-camera-server Lens Search

## Description

HTTP клиент для поиска линз snap-camera-server через `POST /vc/v1/explorer/search`. Используется в UI настройки слота-маски: пользователь вводит название линзы, результаты появляются inline. snap-camera-server не имеет endpoint'а для получения всех линз — только поиск.

## What to do

1. Создать `src/main/snap/search.ts` — `SnapLensSearch`:
   - `search(query: string): Promise<LensResult[]>` — POST к `http://localhost:5645/vc/v1/explorer/search` с `{ query, offset: 0, limit: 50 }`
   - Минимальная длина запроса: 3 символа (возвращать пустой массив для более коротких)
   - Валидировать схему ответа перед использованием (проверять наличие обязательных полей)
   - При недоступности snap-camera-server → возвращать структурированную ошибку `{ type: 'snap_unavailable', message: 'snap-camera-server not found at localhost:5645' }`
2. Добавить тип `LensResult`: `{ id: string; name: string; iconUrl?: string }`
3. IPC handler в `src/main/ipc/snap.ts`:
   - `ipcMain.handle('snap:search', (event, query) => SnapLensSearch.search(query))`
4. Дебаунс — на стороне renderer (Task 6 реализует только серверную логику)

**Dependencies:** Task 1 (project setup)

**Edge cases:**
- snap-camera-server может вернуть пустой массив (нет результатов) — это не ошибка
- Timeout: если snap-camera-server завис — установить timeout 3 секунды

- Использовать встроенный `fetch` (Node 18+) или `node-fetch`
- Валидация схемы: `item && typeof item.id === 'string' && typeof item.name === 'string'`
- `AbortController` для timeout: `signal: AbortSignal.timeout(3000)`
