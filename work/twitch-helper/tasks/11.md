# Task 11: Slot Management UI

## Required Skills

Before starting, load: /skill:code-writing — [SKILL.md](../../../.claude/skills/code-writing/SKILL.md)

## Description

Implement the full slot management UI for the twitch-helper app: slot creation form, slot card (toggle/delete), and lens search component. This task delivers the primary user-facing interaction surface — without it, no slots can be configured and the app is non-functional.

The main screen (Task 10) renders Snap Camera and Media sections with an "Add slot" button. This task fills in the three components those sections depend on: `SlotCard` (displays a configured slot with enable toggle and delete action), `SlotForm` (multi-step form for creating a slot), and `LensSearch` (inline lens search for Mask type slots). Slot state is persisted to `ConfigStore` via IPC handlers in `src/main/ipc/slots.ts`.

Key constraints from the spec:
- Max 5 slots total across both sections (enforced by `SlotService` in main, but UI must count and disable "Add slot" when limit is reached)
- Three slot types: Mask, Media (Медиафайл), Meme (Рандомный мем)
- Reward picker: choose from existing Twitch rewards (`only_manageable_rewards=true`) or create new with name / cost (points) / cooldown
- Toggle enable/disable calls IPC; delete calls IPC and removes card from list

## What to do

1. **Implement `SlotCard.tsx`**
   Create a card component that displays a configured slot. It shows: slot type badge, reward title, type-specific summary (lens name for Mask, file path for Media, folder path for Meme), an enable/disable toggle switch, and a delete button. Toggle click sends `slots:toggle` IPC with `{ id, enabled }`. Delete click sends `slots:delete` IPC with slot id and removes the card from the parent list via callback. Disabled state dims the card body but keeps the toggle accessible.

2. **Implement `SlotForm.tsx`**
   Create a modal/panel form with the following steps rendered in sequence (not wizard tabs — show fields progressively as selections are made):
   - **Type picker:** radio/button group — Mask / Media (Медиафайл) / Meme (Рандомный мем)
   - **Reward selector:** two-option toggle — "Выбрать существующую" / "Создать новую"
     - Existing: dropdown populated by `rewards:list` IPC result (displays `rewardTitle`, stores `rewardId`)
     - New: text input (name), number input (cost in points), number input (cooldown in minutes). On form submit calls `rewards:create` IPC.
   - **Type-specific fields** (rendered only when type is selected):
     - Mask: `LensSearch` component (see below) + text input for hotkey (e.g. `ctrl+shift+1`) + static tooltip block: *"Как назначить hotkey в Snap Camera: Settings → Hotkeys → выберите линзу"*
     - Media: file path field with "Обзор..." button that calls `dialog:openFile` IPC and writes absolute path into field
     - Meme: folder path field with "Обзор..." button that calls `dialog:openFolder` IPC and writes absolute path into field
   - **Save button:** validates that all required fields are filled, then calls `slots:create` IPC with the full slot object; on success closes form and triggers parent refresh of slot list.

3. **Implement `LensSearch.tsx`**
   Inline search component for lens selection inside Mask slot form. Input field triggers debounced (300ms) search when value length >= 3 characters by calling `snap:search` IPC with the query string. Displays results as a dropdown list below the input. Each result shows lens name and a thumbnail if available. Clicking a result sets `lensId` and `lensName` on the parent form and closes the dropdown. When snap-camera-server is unreachable the IPC returns an error — show inline message: *"snap-camera-server не найден на localhost:5645. Запустите snap-camera-server перед настройкой маски."*

4. **Wire into `MainScreen.tsx`**
   `MainScreen` (created in Task 10) renders "Добавить слот" buttons and the slot lists. This task fills those in: render `SlotCard` for each slot in the list, show `SlotForm` when "Добавить слот" is clicked, disable the button when total slot count >= 5.

5. **IPC surface expected from main process** (read `src/main/ipc/slots.ts` before implementing to confirm exact channel names):
   - `slots:list` → returns `Slot[]`
   - `slots:create` → accepts slot payload, returns saved `Slot`
   - `slots:delete` → accepts slot id, returns void
   - `slots:toggle` → accepts `{ id: string, enabled: boolean }`, returns updated `Slot`
   - `rewards:list` → returns `{ rewardId: string, rewardTitle: string }[]`
   - `rewards:create` → accepts `{ name, cost, cooldownMinutes }`, returns `{ rewardId, rewardTitle }`
   - `snap:search` → accepts `{ query: string }`, returns `{ lensId: string, lensName: string }[]` or error
   - `dialog:openFile` → returns `string | null` (absolute path)
   - `dialog:openFolder` → returns `string | null` (absolute path)

### IPC channel names
Read `src/main/ipc/slots.ts` before writing any IPC calls. If the channel names differ from those listed in "What to do", use the actual names from the file. Do not assume — verify.

### Electron security context
`nodeIntegration: false`, `contextIsolation: true`, `sandbox: true` are set per Decision 9. All IPC calls go through `window.electronAPI` (exposed by preload via `contextBridge.exposeInMainWorld`). Never use `require('electron')` in renderer code.

### Slot type grouping in MainScreen
Snap Camera section: only `type === 'mask'` slots. Media section: `type === 'media'` and `type === 'meme'` slots. The 5-slot cap is across both sections combined.

### Reward create: orphan handling
If `rewards:create` IPC succeeds but `slots:create` then fails, the Twitch reward is left orphaned. This is handled in the main process (`SlotService` rolls back by deleting the reward on config write failure — see tech-spec Task 5). The UI only needs to surface the error message returned from `slots:create`.

### Hotkey format
Hotkey input for Mask slot stores a plain string like `ctrl+shift+1`. Validation (parsing to robotjs key sequence) is done in the main process (`HotkeyParser`). The UI should accept any non-empty string — do not implement client-side hotkey parsing.

### Edge cases
- `rewards:list` returns empty array → show "Нет доступных наград. Создайте новую." instead of empty dropdown.
- File/folder dialog cancelled (returns null) → leave path field empty, do not overwrite previously entered value.
- `slots:create` fails with max-5 error → show inline error "Достигнут лимит 5 слотов".
- `snap:search` called while previous search is in-flight → cancel previous by ignoring its result (compare query string on resolve).

### Dependencies
This task has no new npm package dependencies. All UI should be built with React and whatever CSS/component library was set up in Task 1.
