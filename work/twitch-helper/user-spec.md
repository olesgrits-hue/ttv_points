# User Spec: Twitch Helper

## Что делаем

Портативное Windows-приложение для Twitch-стримера. Стример настраивает до 5 слотов, каждый привязан к награде за баллы канала. При выкупе награды зрителем приложение выполняет действие: переключает маску в Snap Camera, воспроизводит конкретный медиафайл или случайный файл из папки через OBS Browser Source.

## Зачем

Сервис SoundAlerts, который позволял воспроизводить медиа на стриме за баллы канала, заблокирован РКН и перестал работать. Готовых отечественных альтернатив нет. Стример также хочет интерактивную смену масок Snap Camera за баллы — этого не умеет ни один известный сервис. Приложение решает обе задачи локально, без зависимости от внешних сервисов.

## Как должно работать

### Первый запуск
1. Пользователь запускает `twitch-helper.exe`
2. Видит экран с кнопкой "Войти через Twitch"
3. Нажимает кнопку — приложение открывает браузер на странице авторизации Twitch (OAuth Authorization Code + PKCE, scopes: `channel:read:redemptions`, `channel:manage:redemptions`)
4. Разрешает доступ — браузер закрывается, приложение показывает главный экран
5. Токен сохраняется в `auth.json` рядом с `.exe` — при следующих запусках логин не нужен
6. Если токен истёк или отозван — приложение автоматически показывает экран логина вместо главного

### Главный экран
Экран разделён на две секции: **Snap Camera** и **Media**. Каждая секция содержит слоты. В нижней части — лог событий.

Строка лога: `[время] [ник зрителя] [название награды] 🟢/🔴`
При наведении на 🔴 — всплывающий тултип с текстом ошибки.

URL для OBS Browser Source отображается в верхней части экрана текстом (например `http://localhost:7891/overlay`).

### Добавление слота
1. Нажимает "Добавить слот" в нужной секции
2. Выбирает тип: **Маска** / **Медиафайл** / **Рандомный мем**
3. Выбирает или создаёт награду Twitch:
   - Создать новую: вводит название, стоимость в баллах, cooldown (раз в N минут / раз в час и т.п.)
   - Выбрать существующую: список загружается из Twitch API
4. Заполняет настройки по типу слота:
   - **Маска**: выбирает линзу из списка snap-camera-server + вводит hotkey назначенный этой линзе в Snap Camera *(подсказка в интерфейсе: "Как назначить hotkey в Snap Camera: Settings → Hotkeys → выберите линзу")*
   - **Медиафайл**: указывает путь к файлу через диалог выбора
   - **Рандомный мем**: указывает путь к папке через диалог выбора
5. Сохраняет — слот появляется в списке с toggle вкл/выкл

### Во время стрима

**Медиафайл / Рандомный мем:**
1. Зритель выкупает награду в Twitch
2. Приложение ставит в очередь FIFO
3. Когда подходит очередь — через WebSocket пушит файл на overlay-страницу
4. OBS Browser Source отображает видео: случайная позиция и угол поворота в пределах области, фиксированный размер
5. Видео воспроизводится до конца; overlay-страница сигнализирует приложению через WebSocket что воспроизведение завершено
6. После сигнала (или по таймауту 120 сек если overlay не подключён) — Twitch API: `fulfill` (баллы списаны), запись в лог 🟢
7. При ошибке (файл не найден, папка пустая) — Twitch API: `cancel` (баллы возвращены), запись в лог 🔴

**Маска Snap Camera:**
1. Зритель выкупает награду
2. Приложение ставит в очередь FIFO
3. Когда подходит очередь — симулирует нажатие hotkey через Windows SendInput API
4. Snap Camera переключает маску
5. Приложение ждёт 30 секунд
6. Симулирует нажатие "снять маску" (hotkey без маски)
7. Twitch API: `fulfill`, запись в лог 🟢

**Если несколько выкупов одновременно:** FIFO-очередь, каждый ждёт своей очереди.

### Потеря соединения
При обрыве WebSocket с Twitch — показывает уведомление "Соединение потеряно, переподключаюсь..." и автоматически переподключается. Настройки слотов при этом не сбрасываются.

## Критерии приёмки

- [ ] Логин через Twitch работает, токен сохраняется в `auth.json` и при следующем запуске приложение стартует уже авторизованным
- [ ] Создание новой награды Twitch из приложения с настройкой cooldown
- [ ] Выбор существующей награды Twitch из списка
- [ ] Слот типа **Маска**: при выкупе hotkey симулируется, маска в Snap Camera переключается, через 30 сек снимается
- [ ] Слот типа **Медиафайл**: при выкупе видео воспроизводится в OBS Browser Source со случайной позицией и углом, фиксированным размером
- [ ] Слот типа **Рандомный мем**: при выкупе случайный файл из папки воспроизводится в OBS Browser Source со случайной позицией и углом, фиксированным размером
- [ ] Несколько одновременных выкупов обрабатываются в порядке очереди (FIFO)
- [ ] Успешный выкуп → `fulfill` (баллы списаны) + 🟢 в логе
- [ ] Ошибочный выкуп (файл не найден, папка пустая) → `cancel` (баллы возвращены) + 🔴 в логе с тултипом
- [ ] Toggle вкл/выкл слота работает без удаления настроек
- [ ] При потере соединения с Twitch — уведомление + авто-реконнект без потери очереди
- [ ] Overlay прозрачный когда нет активного воспроизведения
- [ ] `config.json` и `auth.json` сохраняются между перезапусками exe
- [ ] Слот типа **Маска**: после 30 секунд приложение симулирует "снять маску" (отдельный hotkey без маски), лицо возвращается без маски
- [ ] Список линз для слота-маски подгружается из snap-camera-server API; если API недоступен — показывает ошибку с инструкцией
- [ ] В UI при настройке слота-маски есть подсказка как назначить hotkey в Snap Camera
- [ ] При выборе существующей награды — доступны только награды созданные этим приложением (иначе fulfill/cancel невозможен)

## Ограничения

- **Платформа:** Windows 10+ (64-bit), без требований к правам администратора
- **Портативность:** приложение работает из любой папки без установки, все данные (`config.json`, `auth.json`) хранятся рядом с `.exe`; пути к медиафайлам хранятся абсолютными — при переносе приложения в другую папку пути к файлам нужно переназначить вручную
- **Twitch:** только для Twitch Affiliate / Partner (требование Channel Points API)
- **Snap Camera:** пользователь должен самостоятельно запустить snap-camera-server и назначить hotkeys в Snap Camera до настройки маски-слотов
- **OBS:** пользователь добавляет Browser Source в OBS вручную по URL из приложения
- **MVP:** максимум 5 слотов (секция Snap Camera + секция Media), без автообновления, без импорта/экспорта настроек

## Риски

- **Snap Camera hotkey не срабатывает:** Snap Camera потеряла фокус или не запущена. **Митигация:** документируем это в подсказке; fulfill всё равно происходит, т.к. это вина пользователя (Snap Camera не настроена).
- **Twitch меняет Channel Points API:** периодически меняются условия доступа. **Митигация:** личное использование, готов адаптироваться; EventSub — стабильный стандарт.
- **Большой размер exe (Electron ~150MB):** неудобно для скачивания. **Митигация:** приемлемо для личного инструмента.
- **snap-camera-server не запущен при старте приложения:** список линз не загрузится. **Митигация:** показывать ошибку в UI с инструкцией.
- **snap-camera-server может не иметь API для списка линз:** эндпоинт нужно верифицировать в ходе тех-спека. **Митигация:** если API нет — откат к ручному вводу ID линзы (вариант B).

## Технические решения

- Мы решили использовать **Electron (TypeScript)**, потому что он нативно поддерживает браузерный overlay (тот же движок что и OBS Browser Source), имеет зрелую экосистему и позволяет быстро разработать desktop + web-части в одном проекте.
- Мы решили переключать маски через **Windows SendInput** (симуляция hotkey), потому что snap-camera-server не предоставляет API для смены активной маски — он только отдаёт файлы линз.
- Мы решили хранить конфиг в **`config.json` рядом с exe**, потому что это гарантирует сохранность настроек при ручной замене exe на новую версию.
- Мы решили **не делать автообновление в MVP**, потому что это личный инструмент и пользователь готов заменять exe вручную.
- Мы решили **не ограничивать очередь** в приложении, потому что лимиты (cooldown, макс. на пользователя) настраиваются на уровне Twitch Channel Point reward.
- Мы решили, что **маска снимается через 30 секунд** автоматически, потому что это предсказуемый и управляемый UX без зависимости от следующего выкупа.
- Мы решили использовать **`nut-js` или `robotjs`** для симуляции hotkey вместо прямого вызова Windows SendInput через FFI, потому что эти библиотеки абстрагируют платформу и проще в интеграции с Node.js/Electron.
- Мы решили не использовать **obs-websocket** для overlay, потому что Browser Source проще настраивается для пользователя (один URL) и не требует установки OBS-плагина.

## Тестирование

**Unit-тесты:** делаются всегда, не обсуждаются.

**Интеграционные тесты:** делаем — проверяем подключение к Twitch EventSub (с Twitch CLI для симуляции событий) и работу локального HTTP+WebSocket overlay-сервера.

**E2E тесты:** не делаем — требуют реального стрима и OBS, автоматизация избыточна для личного инструмента.

## Как проверить

### Агент проверяет

| Шаг | Инструмент | Ожидаемый результат |
|-----|-----------|-------------------|
| 1. Запустить overlay-сервер | bash: `node overlay-server.js` | Сервер слушает на localhost:7891 |
| 2. Открыть overlay в браузере | curl: `GET http://localhost:7891/overlay` | HTML-страница с прозрачным фоном |
| 3. Отправить тестовое событие воспроизведения | bash: WebSocket push `{type: "play", file: "test.mp4"}` | Страница отображает видео-элемент |
| 4. Симулировать Twitch redemption | Twitch CLI: `twitch event trigger channel-points-custom-reward-redemption-add` | Событие появляется в логе приложения |
| 5. Проверить config.json после сохранения слота | bash: `cat config.json` | Файл содержит настройки слота |

### Пользователь проверяет
- Войти через Twitch → убедиться что в Dashboard появилась тестовая награда, созданная из приложения
- Выкупить награду с другого аккаунта → видео воспроизводится в OBS с случайной позицией
- Выкупить сломанный слот (удалить файл) → баллы вернулись, в логе 🔴 с описанием ошибки
- Выкупить маску → Snap Camera переключила маску, через 30 сек вернулась к лицу без маски
- Закрыть приложение → overlay в OBS стал прозрачным
